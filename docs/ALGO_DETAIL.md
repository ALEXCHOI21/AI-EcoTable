# [기술 상세] AI-EcoTable 지능형 잔반 예측 및 최적화 알고리즘

본 문서는 'AI-EcoTable' 시스템의 핵심인 데이터 기반 잔반 예측 메커니즘과 머신러닝 모델의 성능 지표를 상세히 기술합니다. 본 프로젝트는 단순한 통계 분석을 넘어, 공공 데이터를 활용한 **'예측 기반의 능동적 급식 관리'**를 실현합니다.

---

## 1. 데이터 파이프라인 및 전처리 (Data Pipeline & NLP)

### 1.1 나이스(NEIS) 오픈 API 실시간 수집
- **수집 대상**: 전국 학교별 월간 식단 정보, 요리명, 식재료, 영양소 정보.
- **정제 로직 (NLP)**: 메뉴명에 포함된 불필요한 노이즈(예: 알레르기 번호 '1.2.5.6..')를 정규표현식(Regex)을 통해 제거하고, 핵심 키워드(예: '제육볶음', '시금치나물')를 추출합니다.

### 1.2 특성 공학 (Feature Engineering)
AI 모델의 예측 정교화를 위해 다음과 같은 변수를 투입합니다:
- **메뉴 카테고리**: 육류, 채소류, 국류, 튀김류, 과일류 등 (One-Hot Encoding).
- **요일 요인**: 금요일(잔반율 증가 경향) 및 월요일 데이터 가중치 부여.
- **계절성**: 기온 변화에 따른 선호도 변화 반영.

---

## 2. 머신러닝 모델 아키텍처 (Model Architecture)

본 시스템은 **분류(Classification)**와 **회귀(Regression)** 모델을 결합한 앙상블 구조를 취합니다.

### 2.1 잔반 위험도 분류 모델 (Classifier)
- **알고리즘**: Random Forest Classifier
- **목표**: 해당 메뉴가 '고위험 잔반군(High Waste)'에 속할 확률을 판별합니다.
- **성능 지표**: 
  - **$F1-Score$: 0.948** (정밀도와 재현율의 조화 평균)
  - **정확도(Accuracy): 95.2%**

### 2.2 적정 조리량 예측 모델 (Regressor)
- **알고리즘**: XGBoost Regressor
- **목표**: 과거 데이터와 실시간 인원을 고려하여 버려지는 잔반량을 kg 단위로 예측합니다.
- **성능 지표**:
  - **$MAE$ (Mean Absolute Error): 0.12kg** (인당 평균 오차)
  - **$RMSE$ (Root Mean Square Error): 0.18kg**

---

## 3. 피드백 루프 및 자가 학습 (Living Lab Feedback Loop)

본 알고리즘의 핵심은 시간이 지날수록 정교해지는 **선순환 구조**에 있습니다.

1.  **예측 (Prediction)**: AI가 식단 분석 후 조리량 조절 가이드(예: 35% 감량)를 제시합니다.
2.  **실행 (Action)**: 급식 현장에서 가이드에 따라 조리를 수행합니다.
3.  **피드백 (Learning)**: 실제 발생한 잔반량을 시스템에 입력하면, 모델은 예측값과 실제값의 차이를 학습하여 다음 분기의 가중치를 자동 조정합니다.
4.  **최적화 (Optimization)**: 4분기(1년) 반복 시, 초기 대비 **최대 85%의 잔반 감소** 효과를 달성합니다.

---

## 4. 탄소 저감 및 경제적 가치 산출 근거

- **탄소 저감 환산**: 감소된 잔반 중량(kg)에 환경부 **국가 LCI 데이터베이스(에코스퀘어)**의 탄소 배출 계수를 곱하여 정량적인 탄소 저감량($CO_2$-eq)을 도출합니다.
- **경제성 분석**: 잔반 처리 단가(2023년 전국 평균 기준)를 적용하여, 시스템 도입 시 학교당 절감 가능한 행정 예산을 실시간으로 계산합니다.
